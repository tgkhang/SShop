# Hướng dẫn cài đặt MySQL trên AWS EC2 Instance (Amazon Linux 2023)

> **Lưu ý**: File này hướng dẫn cài đặt MySQL Server trên **Amazon Linux 2023 (AL2023)** EC2 instance.
>
> - Nếu bạn sử dụng **Amazon Linux 2**, vui lòng xem file [03.mysql.ec2.linux.md](./03.mysql.ec2.linux.md)
> - Nếu bạn sử dụng **Ubuntu**, vui lòng xem file [03.mysql.ec2.ubuntu.md](./03.mysql.ec2.ubuntu.md)

## Tổng quan

**Amazon Linux 2023 (AL2023)** là phiên bản mới nhất của Amazon Linux với nhiều thay đổi quan trọng:

- ✅ Sử dụng `dnf` thay vì `yum` (tuy nhiên `yum` vẫn hoạt động như alias)
- ✅ **Không có** lệnh `amazon-linux-extras` (đã bị loại bỏ)
- ✅ **Không cần** EPEL repository (hầu hết packages đã có sẵn)
- ✅ Tương thích với Enterprise Linux 9 (el9) thay vì el7
- ✅ Cập nhật bảo mật tốt hơn và hiệu suất cao hơn

Trên Windows, người ta thường sử dụng **MySQL Installer** để cài đặt MySQL Server và MySQL Workbench với giao diện đồ họa.
Trên Linux, chúng ta sử dụng **command line** để cài đặt MySQL Server và MySQL Client.

## 1. Chuẩn bị môi trường

### 1.1. Cập nhật hệ thống

Trước khi cài đặt MySQL, nên cập nhật toàn bộ packages trên hệ thống:

```bash
sudo dnf update -y
```

**Giải thích lệnh:**

- `sudo`: Chạy lệnh với quyền superuser (root)
- `dnf`: Package manager mới của Amazon Linux 2023 (thay thế yum)
- `update`: Cập nhật tất cả packages đã cài đặt
- `-y`: Tự động trả lời "yes" cho tất cả các câu hỏi xác nhận

### 1.2. Về DNF Package Manager

**DNF** (Dandified YUM) là package manager mới thế hệ kế tiếp của YUM với nhiều cải tiến:

- ✅ Hiệu suất tốt hơn (xử lý dependencies nhanh hơn)
- ✅ Sử dụng ít bộ nhớ hơn
- ✅ API tốt hơn cho developers
- ✅ Tương thích ngược với YUM commands

DNF cho phép người dùng:

- ✅ Cài đặt packages (`dnf install`)
- ✅ Cập nhật packages (`dnf update`)
- ✅ Xóa packages (`dnf remove`)
- ✅ Quản lý dependencies tự động
- ✅ Tìm kiếm packages (`dnf search`)

**Lưu ý:** Bạn vẫn có thể sử dụng `yum` thay cho `dnf` vì Amazon Linux 2023 tạo alias tự động.

## 2. Cài đặt MySQL Server

### 2.1. Tải và cài đặt MySQL Repository

Truy cập <https://dev.mysql.com/downloads/repo/yum/> để tìm phiên bản MySQL mới nhất.

**⚠️ QUAN TRỌNG:** Amazon Linux 2023 tương thích với **Enterprise Linux 9 (el9)**, không phải el7!

```bash
# Tải và cài đặt MySQL repository package cho EL9
sudo dnf install https://dev.mysql.com/get/mysql80-community-release-el9-1.noarch.rpm -y
```

**Giải thích:**

- Link này tải xuống MySQL 8.0 Community Repository cho **Enterprise Linux 9**
- `el9`: Phiên bản dành cho EL9 (tương thích với AL2023)
- `.noarch.rpm`: File package không phụ thuộc vào kiến trúc hệ thống (32-bit hay 64-bit)
- `-y`: Tự động xác nhận cài đặt

### 2.2. Import GPG Key

Để đảm bảo tính xác thực của MySQL packages, import GPG key chính thức:

```bash
sudo rpm --import https://repo.mysql.com/RPM-GPG-KEY-mysql-2023
```

**Giải thích:**

- `rpm`: Red Hat Package Manager
- `--import`: Import GPG public key
- GPG key đảm bảo packages được tải từ nguồn chính thức và không bị giả mạo

### 2.3. Cài đặt MySQL Community Server

```bash
sudo dnf install mysql-community-server -y
```

**Giải thích:**

- `mysql-community-server`: Package chính chứa MySQL Server phiên bản community (miễn phí)
- DNF sẽ tự động tải và cài đặt tất cả dependencies cần thiết

**Kiểm tra phiên bản đã cài:**

```bash
mysql --version
```

**Output mẫu:**

```cmd
mysql  Ver 8.0.35 for Linux on x86_64 (MySQL Community Server - GPL)
```

## 3. Khởi động và quản lý MySQL Service

### 3.1. Về systemctl

**systemctl** là công cụ quản lý services trên Linux sử dụng systemd. Các lệnh thông dụng:

| Lệnh | Mô tả |
|------|-------|
| `systemctl start <service>` | Khởi động service |
| `systemctl stop <service>` | Dừng service |
| `systemctl restart <service>` | Khởi động lại service |
| `systemctl status <service>` | Xem trạng thái service |
| `systemctl enable <service>` | Tự động khởi động service khi boot |
| `systemctl disable <service>` | Tắt tự động khởi động |

### 3.2. Khởi động MySQL

```bash
# Khởi động MySQL service
sudo systemctl start mysqld

# Kiểm tra trạng thái
sudo systemctl status mysqld
```

**Giải thích:**

- `mysqld`: MySQL daemon (service name)
- `status`: Hiển thị trạng thái service (running, stopped, failed, etc.)

**Output khi MySQL đang chạy:**

```terminal
● mysqld.service - MySQL Server
   Loaded: loaded (/usr/lib/systemd/system/mysqld.service; enabled; vendor preset: disabled)
   Active: active (running) since Sun 2026-02-23 10:15:30 UTC; 2min ago
   ...
```

### 3.3. Enable auto-start khi reboot (Khuyến nghị)

```bash
sudo systemctl enable mysqld
```

**Giải thích:**

- `enable`: Đảm bảo MySQL tự động khởi động mỗi khi EC2 instance reboot
- Rất quan trọng cho production servers

**Verify enable status:**

```bash
sudo systemctl is-enabled mysqld
```

**Output:** `enabled`

## 4. Lấy mật khẩu root tạm thời

Khi cài đặt lần đầu, MySQL tự động tạo một **mật khẩu tạm thời** cho user `root` và lưu trong log file. (Linux only)

### 4.1. Về lệnh grep và cat

**grep** (Global Regular Expression Print):

- Tìm kiếm văn bản theo pattern trong file
- Syntax: `grep [options] pattern [file]`
- Options thông dụng:
  - `-i`: Không phân biệt hoa thường
  - `-r`: Tìm kiếm đệ quy trong thư mục
  - `-n`: Hiển thị số dòng
  - `-C 3`: Hiển thị 3 dòng context xung quanh kết quả

**cat** (Concatenate):

- Đọc và hiển thị nội dung file
- Syntax: `cat [options] [file]`
- Thường được dùng kết hợp với pipe `|` để xử lý output

### 4.2. Tìm mật khẩu tạm thời

**Cách 1: Sử dụng grep trực tiếp (Khuyến nghị)**

```bash
sudo grep 'temporary password' /var/log/mysqld.log
```

**Cách 2: Sử dụng cat kết hợp pipe**

```bash
sudo cat /var/log/mysqld.log | grep 'temporary password'
```

**Giải thích:**

- Cần `sudo` vì file log thuộc quyền root
- `/var/log/mysqld.log`: Đường dẫn file log của MySQL
- `|` (pipe): Chuyển output của lệnh bên trái làm input cho lệnh bên phải

**Output mẫu:**

```
2026-02-23T10:15:30.123456Z 6 [Note] [MY-010454] [Server] A temporary password is generated for root@localhost: kR&5tP#xN9qL
```

**Mật khẩu trong ví dụ này là:** `kR&5tP#xN9qL`

## 5. Đăng nhập và đổi mật khẩu root

### 5.1. Đăng nhập MySQL

```bash
mysql -u root -p
```

**Giải thích:**

- `mysql`: MySQL client command
- `-u root`: Đăng nhập với username là `root`
- `-p`: Yêu cầu nhập password (sẽ hiện prompt để nhập mật khẩu)

**Prompt sẽ xuất hiện:**

```
Enter password:
```

Nhập mật khẩu tạm thời vừa tìm được ở bước 4 (ví dụ: `kR&5tP#xN9qL`)

**Khi đăng nhập thành công, bạn sẽ thấy:**

```
Welcome to the MySQL monitor.  Commands end with ; or \g.
Your MySQL connection id is 8
Server version: 8.0.35

Copyright (c) 2000, 2023, Oracle and/or its affiliates.

Type 'help;' or '\h' for help. Type '\c' to clear the current input statement.

mysql>
```

### 5.2. Đổi mật khẩu root

Sau khi đăng nhập thành công vào MySQL, chạy lệnh SQL sau:

```sql
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'Demom@123';
```

**Giải thích:**

- `ALTER USER`: Lệnh SQL để thay đổi thông tin user
- `'root'@'localhost'`: User root chỉ được phép kết nối từ localhost
- `IDENTIFIED WITH mysql_native_password`: Sử dụng phương thức xác thực native (tương thích tốt với các ứng dụng cũ)
- `BY 'YourNewPassword123!'`: Mật khẩu mới của bạn

**⚠️ Yêu cầu mật khẩu mạnh:**

MySQL 8.0 có policy mật khẩu mạnh mặc định. Mật khẩu phải có:

- ✅ Ít nhất 8 ký tự
- ✅ Chữ hoa (A-Z)
- ✅ Chữ thường (a-z)
- ✅ Số (0-9)
- ✅ Ký tự đặc biệt (!@#$%^&*)

**Ví dụ mật khẩu hợp lệ:**

- `MyShop2026@Pass`
- `SecureDB#2026!`
- `Tipjs@MySQL123`

**Apply changes:**

```sql
FLUSH PRIVILEGES;
```

**Giải thích:**

- `FLUSH PRIVILEGES`: Reload bảng grant để các thay đổi có hiệu lực ngay lập tức

## 6. Tạo User và Quản lý Quyền (QUAN TRỌNG!)

> **⚠️ BEST PRACTICE:** **KHÔNG BAO GIỜ** sử dụng user `root` cho ứng dụng! User `root` giống như "tổng thống" - chỉ dùng cho quản trị hệ thống. Luôn tạo user riêng cho mỗi ứng dụng/database.

### 6.1. Tại sao không dùng root?

**Lý do bảo mật:**

- ❌ `root` có toàn bộ quyền trên tất cả databases - rất nguy hiểm nếu bị hack
- ❌ Nếu ứng dụng bị SQL injection, hacker có thể xóa toàn bộ server
- ❌ Khó audit và tracking ai làm gì khi tất cả đều dùng chung root
- ✅ Mỗi ứng dụng nên có user riêng với quyền hạn tối thiểu cần thiết

### 6.2. Tạo User cho Local Access (Ứng dụng trên cùng server)

**Đăng nhập MySQL với root:**

```bash
mysql -u root -p
```

**Tạo user chỉ kết nối từ localhost:**

```sql
CREATE USER 'tipjs'@'localhost' IDENTIFIED WITH mysql_native_password BY 'Demo@123';
```

**Giải thích:**

- `CREATE USER`: Lệnh tạo user mới
- `'tipjs'@'localhost'`: User `tipjs` chỉ được phép kết nối từ localhost (cùng server)
- `IDENTIFIED WITH mysql_native_password`: Plugin xác thực (tương thích tốt với Node.js, PHP)
- `BY 'Demo@123'`: Mật khẩu của user

**Cấp quyền cho user trên một database cụ thể:**

```sql
-- Cấp toàn bộ quyền trên database "sampledb"
GRANT ALL PRIVILEGES ON sampledb.* TO 'tipjs'@'localhost';

-- Apply changes
FLUSH PRIVILEGES;
```

**Giải thích:**

- `GRANT ALL PRIVILEGES`: Cấp tất cả quyền (SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, etc.)
- `ON sampledb.*`: Trên database `sampledb` và tất cả tables (`*`)
- `TO 'tipjs'@'localhost'`: Cho user `tipjs` từ localhost

**Các ví dụ cấp quyền khác:**

```sql
-- Cấp quyền trên tất cả databases (thận trọng!)
GRANT ALL PRIVILEGES ON *.* TO 'tipjs'@'localhost';

-- Chỉ cấp quyền đọc/ghi, không cho xóa database
GRANT SELECT, INSERT, UPDATE, DELETE ON sampledb.* TO 'tipjs'@'localhost';

-- Chỉ cấp quyền đọc (read-only)
GRANT SELECT ON sampledb.* TO 'readonly_user'@'localhost';

-- Cấp quyền trên table cụ thể
GRANT ALL PRIVILEGES ON sampledb.users TO 'tipjs'@'localhost';

-- Apply changes
FLUSH PRIVILEGES;
```

**Thoát ra và test đăng nhập với user mới:**

```bash
exit
```

```bash
mysql -u tipjs -p
```

**Nhập mật khẩu:** `Demo@123`

**Kiểm tra quyền:**

```sql
SHOW GRANTS;
USE sampledb;
SHOW TABLES;
```

### 6.3. Tạo User cho Remote Access (Kết nối từ xa)

**Trường hợp sử dụng:**

- Ứng dụng chạy trên server khác (ví dụ: App server riêng, DB server riêng)
- Kết nối từ công cụ quản trị: **DBeaver**, **Navicat**, **MySQL Workbench**, **HeidiSQL**
- Kết nối từ máy local để develop/debug

**Tạo user có thể kết nối từ bất kỳ IP nào:**

```bash
mysql -u root -p
```

```sql
CREATE USER 'tipjs'@'%' IDENTIFIED WITH mysql_native_password BY 'Demo@123';
```

**Giải thích:**

- `'tipjs'@'%'`: `%` là wildcard - cho phép kết nối từ **bất kỳ IP nào**
- Lưu ý: User `'tipjs'@'localhost'` và `'tipjs'@'%'` là **2 users khác nhau** trong MySQL!

**Cấp quyền cho remote user:**

```sql
-- Cấp quyền trên database cụ thể
GRANT ALL PRIVILEGES ON sampledb.* TO 'tipjs'@'%';

-- Hoặc cấp trên tất cả databases
GRANT ALL PRIVILEGES ON *.* TO 'tipjs'@'%';

-- Apply changes
FLUSH PRIVILEGES;
```

**Verify user đã được tạo:**

```sql
-- Xem danh sách tất cả users
SELECT User, Host FROM mysql.user;

-- Xem quyền của user cụ thể
SHOW GRANTS FOR 'tipjs'@'%';
SHOW GRANTS FOR 'tipjs'@'localhost';
```

**Output mẫu:**

```
+-------+-----------+
| User  | Host      |
+-------+-----------+
| root  | localhost |
| tipjs | localhost |
| tipjs | %         |
+-------+-----------+
```

### 6.4. Cấu hình MySQL cho Remote Connection

**⚠️ Lưu ý:** Để remote connection hoạt động, cần cấu hình thêm:

**Bước 1: Sửa MySQL config để listen trên tất cả IP**

```bash
sudo nano /etc/my.cnf
```

**Thêm hoặc sửa trong section `[mysqld]`:**

```ini
[mysqld]
bind-address = 0.0.0.0
```

**Lưu file (Ctrl+O, Enter, Ctrl+X) và restart MySQL:**

```bash
sudo systemctl restart mysqld
```

**Bước 2: Mở port 3306 trong AWS Security Group**

1. Vào **EC2 Dashboard** → **Instances** → Chọn instance
2. Tab **Security** → Click vào **Security Group**
3. **Inbound rules** → **Edit inbound rules**
4. **Add rule:**
   - Type: **MySQL/Aurora**
   - Port: **3306**
   - Source:
     - **My IP** (khuyến nghị - chỉ IP của bạn)
     - **Custom** (nhập IP cụ thể của app server)
     - **0.0.0.0/0** (tất cả IP - ⚠️ không an toàn!)
5. **Save rules**

### 6.5. Test Remote Connection

**Từ máy local (Windows PowerShell):**

```bash
mysql -h 47.129.32.211 -u tipjs -p
```

**Giải thích:**

- `-h 47.129.32.211`: IP public của EC2 instance (không dùng localhost!)
- `-u tipjs`: Username đã tạo
- `-p`: Nhập password

**⚠️ Lỗi thường gặp:**

```
ERROR 2003 (HY000): Can't connect to MySQL server on '47.129.32.211' (10060)
```

**Nguyên nhân và cách fix:**

1. ❌ Security Group chưa mở port 3306 → Xem bước 6.4
2. ❌ MySQL bind-address = 127.0.0.1 → Đổi thành 0.0.0.0
3. ❌ User không có quyền remote (`@'localhost'`) → Tạo user với `@'%'`
4. ❌ Firewall trên Linux chặn → Xem phần troubleshooting

### 6.7. Tại sao từ localhost không dùng được IP?

Khi bạn SSH vào EC2 instance và cố gắng kết nối qua IP public:

```bash
# Từ bên trong EC2 instance
mysql -h 47.129.32.211 -u tipjs -p
```

**Sẽ bị lỗi hoặc rất chậm** vì:

- IP public được AWS route qua Internet gateway
- Từ localhost thì connection sẽ đi ra Internet rồi quay lại
- AWS có thể block connection này (hairpin NAT)

**✅ Từ localhost trên EC2, luôn dùng:**

```bash
mysql -u tipjs -p
# hoặc
mysql -h localhost -u tipjs -p
# hoặc  
mysql -h 127.0.0.1 -u tipjs -p
```

**IP public chỉ dùng để kết nối TỪ BÊN NGOÀI (máy local, app server khác).**

### 6.8. Best Practices cho User Management

| Scenario | User Host | Quyền | Ghi chú |
|----------|-----------|-------|---------|
| App trên cùng server | `@'localhost'` | Chỉ database cần thiết | An toàn nhất |
| App trên server khác | `@'app_server_ip'` | Chỉ database cần thiết | Nên dùng IP cố định |
| Development/Testing | `@'%'` | Database dev/test | Chỉ dùng cho môi trường dev |
| Read-only reporting | `@'%'` | SELECT only | Cho các công cụ BI |
| Admin user | `@'your_ip'` | ALL PRIVILEGES | Chỉ dùng khi cần |

**⚠️ Cảnh báo bảo mật:**

- **KHÔNG** tạo user `'admin'@'%'` với `ALL PRIVILEGES ON *.*` trên production!
- **LUÔN** giới hạn quyền theo nguyên tắc "least privilege" (quyền tối thiểu cần thiết)
- **SỬ DỤNG** SSL/TLS cho remote connections trên production
- **THƯỜNG XUYÊN** đổi password và audit user access

## 7. Import dữ liệu từ file SQL

### 6.1. Chuẩn bị file SQL

Giả sử bạn có file `mysqlsampledatabase.sql` trên máy local.

### 6.2. Về lệnh SCP

**SCP** (Secure Copy Protocol):

- Copy file an toàn giữa local và remote server qua SSH
- Syntax: `scp [options] source destination`
- Options thông dụng:
  - `-i <key_file>`: Sử dụng private key để xác thực
  - `-r`: Copy đệ quy (cho thư mục)
  - `-P <port>`: Chỉ định port SSH (mặc định 22)
  - `-v`: Verbose mode (hiển thị chi tiết quá trình)

### 6.3. Copy file SQL từ local lên EC2

**⚠️ QUAN TRỌNG:** Lệnh `scp` phải chạy từ **máy local (Windows)**, KHÔNG phải từ bên trong EC2 instance!

**Nếu đang SSH trong EC2, hãy thoát ra trước:**

```bash
exit
```

**Trên máy Windows (PowerShell hoặc Command Prompt):**

```powershell
# Chạy từ thư mục chứa file .pem (ví dụ: C:\Users\trang\.ssh)
cd ~/.ssh
scp -i "mysql-shopdev-tipjs.pem" A:\REPO\SShop\be\nodejs-be\docs\aws\mysqlsampledatabase.sql ec2-user@ec2-47-129-32-211.ap-southeast-1.compute.amazonaws.com:/home/ec2-user/
```

**Hoặc sử dụng đường dẫn đầy đủ:**

```powershell
scp -i "C:\Users\trang\.ssh\mysql-shopdev-tipjs.pem" A:\REPO\SShop\be\nodejs-be\docs\aws\mysqlsampledatabase.sql ec2-user@ec2-47-129-32-211.ap-southeast-1.compute.amazonaws.com:/home/ec2-user/
```

## 8. Các lệnh MySQL thông dụng

### 8.1. Database Operations

```sql
-- Xem danh sách databases
SHOW DATABASES;

-- Tạo database mới
CREATE DATABASE mydb;

-- Sử dụng database
USE mydb;

-- Xóa database
DROP DATABASE mydb;

-- Xem database đang sử dụng
SELECT DATABASE();
```

### 8.2. Table Operations

```sql
-- Xem danh sách tables
SHOW TABLES;

-- Xem cấu trúc bảng
DESCRIBE table_name;
-- hoặc
SHOW CREATE TABLE table_name;

-- Tạo bảng mới
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Xóa bảng
DROP TABLE table_name;
```

### 8.3. User Management

```sql
-- Xem danh sách users
SELECT User, Host FROM mysql.user;

-- Xem quyền của user
SHOW GRANTS FOR 'username'@'host';

-- Tạo user mới
CREATE USER 'newuser'@'localhost' IDENTIFIED BY 'password';

-- Cấp quyền
GRANT ALL PRIVILEGES ON database_name.* TO 'username'@'host';

-- Thu hồi quyền
REVOKE ALL PRIVILEGES ON database_name.* FROM 'username'@'host';

-- Xóa user
DROP USER 'username'@'host';

-- Apply changes
FLUSH PRIVILEGES;
```

### 8.4. Query Information

```sql
-- Xem version MySQL
SELECT VERSION();

-- Xem trạng thái server
SHOW STATUS;

-- Xem các process đang chạy
SHOW PROCESSLIST;

-- Xem biến cấu hình
SHOW VARIABLES;
SHOW VARIABLES LIKE 'max_connections';

-- Xem size của databases
SELECT 
    table_schema AS 'Database',
    ROUND(SUM(data_length + index_length) / 1024 / 1024, 2) AS 'Size (MB)'
FROM information_schema.tables
GROUP BY table_schema;
```

## 9. Backup và Restore

### 9.1. Backup Database

**Backup single database:**

```bash
mysqldump -u root -p database_name > backup_file.sql
```

**Backup all databases:**

```bash
mysqldump -u root -p --all-databases > all_databases_backup.sql
```

**Backup với compressed (gzip):**

```bash
mysqldump -u root -p database_name | gzip > backup_file.sql.gz
```

**Giải thích:**

- `mysqldump`: Công cụ backup của MySQL
- `-u root -p`: Username và yêu cầu nhập password
- `>`: Redirect output ra file
- `| gzip`: Nén file bằng gzip

### 9.2. Restore Database

**Restore từ file .sql:**

```bash
mysql -u root -p database_name < backup_file.sql
```

**Restore từ file compressed:**

```bash
gunzip < backup_file.sql.gz | mysql -u root -p database_name
```

### 9.3. Tự động backup với Cron

**Tạo bash script:**

```bash
nano ~/mysql_backup.sh
```

**Nội dung script:**

```bash
#!/bin/bash
TIMESTAMP=$(date +"%Y%m%d_%H%M%S")
BACKUP_DIR="/home/ec2-user/mysql_backups"
MYSQL_USER="root"
MYSQL_PASS="YourPassword"
DATABASE="sampledb"

mkdir -p $BACKUP_DIR
mysqldump -u $MYSQL_USER -p$MYSQL_PASS $DATABASE | gzip > $BACKUP_DIR/${DATABASE}_${TIMESTAMP}.sql.gz

# Xóa backup cũ hơn 7 ngày
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete
```

**Cấp quyền execute:**

```bash
chmod +x ~/mysql_backup.sh
```

**Setup cron job (backup hàng ngày lúc 2AM):**

```bash
crontab -e
```

**Thêm dòng:**

```
0 2 * * * /home/ec2-user/mysql_backup.sh
```

## 10. Troubleshooting

### 10.1. Không thể khởi động MySQL

**Kiểm tra logs:**

```bash
sudo tail -f /var/log/mysqld.log
```

**Kiểm tra status chi tiết:**

```bash
sudo systemctl status mysqld -l
```

### 10.2. Quên mật khẩu root

**Stop MySQL:**

```bash
sudo systemctl stop mysqld
```

**Start MySQL ở safe mode:**

```bash
sudo mysqld_safe --skip-grant-tables &
```

**Đăng nhập không cần password:**

```bash
mysql -u root
```

**Reset password:**

```sql
FLUSH PRIVILEGES;
ALTER USER 'root'@'localhost' IDENTIFIED BY 'NewPassword123!';
FLUSH PRIVILEGES;
exit;
```

**Kill mysqld_safe và restart bình thường:**

```bash
sudo pkill mysqld
sudo systemctl start mysqld
```

### 10.3. Lỗi "ERROR 1819: Your password does not satisfy the current policy"

**Xem policy hiện tại:**

```sql
SHOW VARIABLES LIKE 'validate_password%';
```

**Tạm thời giảm yêu cầu (không khuyến nghị production):**

```sql
SET GLOBAL validate_password.policy=LOW;
SET GLOBAL validate_password.length=6;
```

### 10.4. Không thể kết nối remote

**Checklist:**

1. ✅ MySQL có bind-address = 0.0.0.0
2. ✅ User có quyền truy cập từ remote (`'user'@'%'`)
3. ✅ AWS Security Group đã mở port 3306
4. ✅ Firewall trên EC2 instance (nếu có):

```bash
# Kiểm tra firewall
sudo firewall-cmd --list-all

# Mở port 3306 (nếu firewall đang bật)
sudo firewall-cmd --permanent --add-port=3306/tcp
sudo firewall-cmd --reload
```
